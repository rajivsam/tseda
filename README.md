# Time Series EDA (`tseda`)

A simple tool for the exploratory analysis of your time series data.

## Why is Exploratory Data Analysis (EDA) for Time Series Important?

Time series data is ubiquitous, generated by everything from personal devices to enterprise applications. Before diving into complex tasks like forecasting, classification, or data mining, a thorough exploratory analysis is a critical first step.

An effective EDA helps you:

*   **Build Better Models:** By understanding the sources of variation in your time series, you can develop more accurate and robust forecasting models.
*   **Discover Predictive Features:** For classification tasks, an exploratory analysis can reveal the most useful features for predicting a specific label.
*   **Understand Structural Characteristics:** Gain insights into the fundamental nature of your time series data, which is essential for any application.

`tseda` provides a set of analytical tools to understand the nature and structural characteristics of your regularly sampled time series data (e.g., monthly sales, daily server load), setting you up for success in your broader application development.

## Core Features

*   **Change Point Detection:** Automatically segment your time series into different regimes based on statistical shifts in the data.
*   **Time Series Decomposition:** Analyze the components of variation (e.g., trend, seasonality) within each segment using Singular Spectrum Analysis (SSA).
*   **Visualization of Time series** Use the tools like smoothers and density plots to analyze your time series. There are more features coming shortly.
*   **Knowledge Base Integration:** Capture your findings in a `kmds` knowledge base for collaborative and incremental analysis.
*   **LLM-Powered Summarization:** Use Large Language Models (LLMs) to generate concise summaries of your time series characteristics and save them to your knowledge base.

## The `tseda` Analysis Workflow

`tseda` follows a structured approach to uncover insights from your data:

1.  **Segmentation:** The tool first identifies significant *change points* in the time series, segmenting the data into distinct periods of consistent behavior. If no change points are found, the entire series is treated as a single segment.
2.  **Decomposition:** Within each segment, `tseda` uses Singular Spectrum Analysis (SSA) to decompose the time series into its constituent parts, such as trend and seasonality. This helps you understand what drives the variation in your data. For example, you might find that the trend accounts for most of the variation, while seasonality has a minor impact.
3.  **Summarization & Knowledge Capture:** Finally, you can consolidate the findings from the segmentation and decomposition steps. `tseda` leverages LLMs to help you create clear, concise descriptions of the observed behavior and capture this structured knowledge in a `kmds` knowledge base.

## Collaborative Analysis with `kmds`

`tseda` is integrated with `kmds` (Knowledge Management for Data Science), enabling a powerful, collaborative workflow:

*   **Incremental Knowledge Building:** Add your understanding of the data incrementally as your analysis evolves and as new data becomes available.
*   **LLM-Powered Research:** Use a built-in chat interface to research your findings with a Large Language Model.
*   **Export for RAG:** Export the structured data from your knowledge graph to build features for applications like Retrieval Augmented Generation (RAG).
*   **Automated Summaries:** A template is provided to automatically generate and log concise data summaries into `kmds` based on the characteristics you input.

## Getting Started

1.  **Set up Knowledge Repository:** `kmds` uses `git` for knowledge storage. Create a new Git repository for your knowledge base. To collaborate, team members can simply clone this repository.

2.  **Configure API Key:** `tseda` uses the Gemini Large Language Model. Create a `.env` file in the project root and add your API key:
    ```
    GEMINI_API_KEY="YOUR_API_KEY"
    ```

3.  **Installation:** Install the required Python packages.
    ```bash
    pip install -r requirements.txt
    ```

4.  **Launch the App:**
    ```bash
    streamlit run src/user_interface/ts_analyze_ui.py
    ```
